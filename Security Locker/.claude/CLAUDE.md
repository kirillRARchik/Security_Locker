# Project Memory

Instructions here apply to this project and are shared with team members.

## Context

# Описание проекта
- Язык программирования (сервер): Python.
- Фреймворк: Flask.
- Клиент: HTML/Jinja2, CSS, JS.
- База данных: SQLite3.

# Правила кодинга
- Пиши чистый, модульный код с типами
- Всегда следуй принципам DRY и KISS
- Перед внесением изменений изучи существующие компоненты.
- Не создавай новые файлы, если задачу можно решить изменением старых
- Продолжай писать код в том же стиле, который есть у уже имеющегося кода
- Не смешивай роли в рамках архитектуры MVC

# Требования к стилю

Сначала проанализируй существующую кодовую базу и выяви используемые паттерны проектирования, соглашения по именованию, структуру модулей, подход к работе с БД, обработке ошибок и оформлению HTML-шаблонов.

После этого реализуй новую функциональность, строго придерживаясь обнаруженного стиля. Не вводи новые архитектурные подходы, если аналогичная задача уже решена в проекте существующим способом.

# Общение
- Подробно описывай функционал сгенерированного кода.

# Распределение ролей
- Backend-функционал пиши в новые файлы (текущий app.py уже практически перегружен)

# User Flow
Login → Dashboard → Choose Module → Study the material → Pass Quiz → Get Score → Download Certificate → Progress Updated
